# V2A — Voice to Agent

A minimalist native iOS app that turns speech into clean, agent-ready text.

Speak a thought → **Soniox** transcribes it in real time → an **AI provider you choose**
cleans it up → copy the result into ChatGPT, Claude, or any other AI agent.
Built for people who chat with AI agents a lot and find typing too slow.

Pure SwiftUI, iOS 17+. **Bring your own keys (BYOK)** — no backend, no accounts,
no data collection. Every network call goes directly from your device to Soniox
or to your chosen AI provider, using keys stored only in your device Keychain.

## Features

- **Real-time transcription** via Soniox (`stt-rt-v5`), 60+ languages, selectable language hints + hotwords.
- **Five AI providers** for cleanup, pick one and paste your key: Deepseek, Claude (Haiku), Gemini (Flash), OpenAI, Groq. Streaming output.
- **Two cleanup styles** — *Quick* (light polish) and *Deep* (resolves mid-sentence self-corrections, turns spoken enumerations into bullet points), plus two custom prompt slots you can write yourself.
- **Light / dark / system** appearance, **English / 中文 / system** UI language.
- Onboarding, in-app FAQ for getting each provider's key, transcript history (local, optional), share sheet, auto-copy.
- Friendly error handling: out-of-balance / invalid-key / offline / rate-limit all get clear messages (with a top-up link for billing issues).

## Architecture

- `V2A/Models/AppState.swift` — single `@Observable` state machine (`@MainActor`).
- `V2A/Services/`
  - `MicRecorder.swift` — AVAudioEngine + AVAudioConverter → 16 kHz Int16 PCM frames, audio-interruption handling.
  - `SonioxClient.swift` — `URLSessionWebSocketTask` client for Soniox real-time STT.
  - `Providers/` — a `CleanupProvider` abstraction + three low-level clients (OpenAI-compatible, Anthropic, Gemini) shared across the five providers, all streaming-capable.
  - `KeychainStore.swift`, `HotwordsStore.swift`, `HistoryStore.swift`, `PromptDefaults.swift`, `AppError.swift`.
- `V2A/Views/` — SwiftUI screens (main, settings, onboarding, prompt manager, FAQ, about, history, language picker) + `AppTheme.swift` (semantic color tokens, spacing scale, button styles).
- `V2A/Resources/Localizable.xcstrings` — String Catalog (zh-Hans source + English).

## Build

Requires [xcodegen](https://github.com/yonsm/XcodeGen) (`brew install xcodegen`) and Xcode 16+.

```bash
# generate the project + run in the simulator
xcodegen generate
xcodebuild -project V2A.xcodeproj -scheme V2A \
  -destination "platform=iOS Simulator,name=iPhone 17" build
```

Install on a connected device (set your own UDID + team):

```bash
export V2A_DEVICE_ID="<your iPhone UDID>"   # xcrun devicectl list devices
./scripts/build-and-install.sh
```

### Configuration (fork this)

- **Bundle ID**: `com.charlesgxy.v2a` — change in `project.yml`.
- **Team ID**: `SRYLV3Q454` — change to yours in `project.yml`.
- **Deployment target**: iOS 17.0.

`*.xcodeproj` is generated by xcodegen and git-ignored; `project.yml` is the source of truth.

### TestFlight / App Store upload

`scripts/archive-and-upload.sh` does a headless Release archive + upload using an
App Store Connect API key and manual signing. It reads everything from the
environment — **no credentials are committed**:

```bash
export ASC_KEY_ID="<your key id>"
export ASC_ISSUER_ID="<your issuer id>"
# key file expected at ~/.appstoreconnect/private_keys/AuthKey_<ASC_KEY_ID>.p8
export KEYCHAIN_PWD="<login keychain password>"   # optional, to unlock for codesign
./scripts/archive-and-upload.sh
```

`scripts/make_dist_cert.py` and `scripts/make_appstore_profile.py` create a
distribution certificate / App Store provisioning profile via the App Store
Connect API (same env vars) when you need to (re)provision.

## Privacy

No backend, no analytics, no ads. API keys live in the device Keychain; hotwords,
custom styles, and history stay in local storage. When you record, audio is sent
to Soniox with your Soniox key; cleanup text is sent to your chosen AI provider
with that provider's key. Nothing routes through any server owned by this project.
See `V2A/Views/AboutView.swift` for the in-app privacy text, or the hosted
policy at <https://charlesguooo.github.io/v2a-privacy/>.
